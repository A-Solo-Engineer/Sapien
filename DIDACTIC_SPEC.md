## 1. Purpose of This Document

ARCHITECTURE.md describes the Didactic Episode in prose and as an
operational pipeline. This document provides the formal mathematical
foundation underneath that description.

The goal is to define the episode precisely enough that an independent
researcher can:

- Determine whether a given implementation is a valid Didactic Episode
- Formally reason about termination and convergence
- Identify precisely where formal gaps remain

This is a specification, not a proof of correctness.
The open problems in Section 11 document where the specification is
itself incomplete.

## 3. Agent Definitions

**Teacher agent T:**

T is a generative model with access to a knowledge domain.
In Generation 0, T is a frontier LLM.
In subsequent generations, T is a mature Sapien instance.

T must be capable of:
- Presenting conceptual material as bounded chunks c₁, c₂, ..., cₙ
- Answering questions q with responses that include a WHY chain
- Partitioning a topic τ into an ordered sequence of chunks

**Learner agent L:**

L maintains a knowledge graph G and a knowledge gap map.

L must be capable of:
- Evaluating whether a chunk c connects to existing graph G
- Generating questions that point to genuine gaps in G
- Committing new nodes and edges to G
- Computing the Known Unknowns set KU at any moment

**Verifier agent V:**

V evaluates teacher responses for internal consistency and
detectable hallucination. V is itself a probabilistic model
and is not infallible. See LIMITATIONS.md §2.

**Human supervisor H:**

H holds final epistemic authority. H reviews outputs flagged
by V and all SEED node creation events.

## 4. Epistemic State Space

Every concept reachable within topic τ exists in one of four
epistemic states relative to learner L at time t:

```
KNOWN          ∈ KK(t)   — complete WHY chain established
KNOWN_UNKNOWN  ∈ KU(t)   — gap identified, question pending
SEED           ∈ S(t)    — isolated new domain, no connections yet
PENDING        ∈ P(t)    — SEED node with partial connections,
                            awaiting bridge knowledge
```

These sets are mutually exclusive and collectively exhaustive
over the concepts encountered during an episode:

```
KK(t) ∩ KU(t) = ∅
KK(t) ∩ S(t)  = ∅
KU(t) ∩ S(t)  = ∅
```

The Unknown Unknown space UU cannot be enumerated — by definition,
L does not know what is in it. UU(t) shrinks as T introduces
new chunks that expose previously invisible territory.

## 5. The Didactic Episode as a State Transition System

A Didactic Episode E is a tuple:

```
E = ⟨ τ, T, L, V, H, G₀, Δ ⟩
```

Where:

- τ    is the topic of the episode
- T    is the teacher agent
- L    is the learner agent
- V    is the verifier agent
- H    is the human supervisor
- G₀   is the initial knowledge graph (possibly empty for first episode)
- Δ    is the transition function defined below

**State:**

At each step t, the system state is:

```
Σ(t) = ⟨ Gₜ, KK(t), KU(t), S(t), P(t) ⟩
```

**Transition:**

The transition function Δ maps a state and an event to the
next state:

```
Δ : Σ(t) × Event → Σ(t+1)
```

Events are:
- `chunk_presented(c)`   — T presents chunk c to L
- `connected(n)`         — L connects chunk c to existing node n in G
- `partial(n, gap)`      — L connects partially, with identified gap
- `seed_created(s)`      — L creates SEED node s
- `question_asked(q)`    — L generates question q from gap
- `answer_received(a)`   — T responds to q with answer a and WHY chain
- `verified(a, result)`  — V evaluates answer a
- `committed(n)`         — node n is committed to G
- `closure_reached`      — KU(t) = ∅

## 6. Formal Episode Execution

### Initialization

```
t    ← 0
Gₜ   ← G₀
KK   ← { n ∈ G₀ : status(n) = KNOWN }
KU   ← { n ∈ G₀ : status(n) = KNOWN_UNKNOWN }
S    ← { n ∈ G₀ : status(n) = SEED }
P    ← { n ∈ G₀ : status(n) = PENDING }
```

### Main Loop

```
while KU(t) ≠ ∅ or chunks remain in τ:

    c ← T.next_chunk(τ)          — T presents next subtopic chunk
    result ← φ(c, Gₜ)           — L attempts connection to G

    case result of:

        FULL_CONNECTION:
            n ← create_node(c, status=KNOWN)
            establish_why_chain(n, Gₜ)
            Gₜ₊₁ ← Gₜ ∪ {n}
            KK ← KK ∪ {n}

        PARTIAL_CONNECTION:
            n ← create_node(c, status=KNOWN_UNKNOWN)
            Gₜ₊₁ ← Gₜ ∪ {n}
            KU ← KU ∪ {n}
            q ← L.generate_question(n, Gₜ)
            if is_unique(q, Gₜ):
                a ← T.answer(q)
                verified ← V.check(a)
                if verified = FLAGGED:
                    a ← H.review(a)
                commit_why_chain(n, a)
                KU ← KU \ {n}
                KK ← KK ∪ {n with status=KNOWN}

        NO_CONNECTION:
            s ← create_seed_node(c)
            Gₜ₊₁ ← Gₜ ∪ {s}
            S  ← S ∪ {s}
            H.notify_seed(s)
            reward(L, SEED_CREATION_REWARD)

    t ← t + 1
```

### Termination

```
if KU(t) = ∅:
    mark_episode_closed(E)
    for each s in S:
        s.status ← PENDING   — SEED nodes await future episodes
```

## 7. Epistemic Closure — Formal Definition

An episode E reaches epistemic closure at time t if and only if:

```
KU(t) = ∅
```

This means:
- Every chunk presented by T has been classified
- Every identified gap has been closed with a WHY chain
- No unresolved KNOWN_UNKNOWN nodes remain in Gₜ

**What closure does not mean:**

Epistemic closure is local to episode E and topic τ.
It does not mean:
- L has exhausted topic τ — the teacher may have presented only a
  subset of Ω(τ)
- SEED nodes are resolved — they are explicitly deferred to
  future episodes with status PENDING
- The WHY chains are correct — correctness is a separate
  verification concern from closure
- L has no remaining ignorance — Unknown Unknowns are by
  definition invisible until exposed by future teaching

Epistemic closure is a statement about L's known gap list,
not about L's total knowledge.

## 8. Termination Guarantee

**Claim:** Every well-formed Didactic Episode terminates.

**Argument:**

A Didactic Episode terminates when KU(t) = ∅ and no new chunks
remain in τ.

For termination to be guaranteed:

1. The chunk sequence of τ must be finite.
   T partitions τ into a finite ordered sequence of chunks.
   Each chunk is presented exactly once.

2. Each chunk either:
   - Enters KK — removes nothing from KU, adds nothing
   - Enters KU — adds one element, which is then removed
     by question/answer exchange in the same step
   - Enters S — does not affect KU

3. Therefore, KU grows only when chunks produce PARTIAL_CONNECTION
   and shrinks as questions close those gaps.

4. Since the chunk sequence is finite and each question generated
   within a step closes the gap that produced it, KU eventually
   empties and all chunks are exhausted.

**Known weakness in this argument:**

The argument assumes a question successfully closes its gap.
If the teacher's answer is inadequate — flagged by V and rejected
by H — the gap persists. The protocol as specified does not define
a finite retry limit for teacher failure.

This is an open problem. See Section 11, item 1.

## 9. Uniqueness Check for Question Generation

Before L submits question q, a uniqueness check is performed
against the full question history Q of the current episode:

```
is_unique(q, Q, Gₜ):

    for each q' in Q:
        sim ← σ(q, q')          — compute semantic similarity
        if sim > θ_similarity:
            return FALSE         — question is a rephrasing

    gap_exists ← ∃ n in KU(t) : q points to gap of n
    if not gap_exists:
        return FALSE             — question does not point to a real gap

    return TRUE
```

Where:
- σ(q, q') is a semantic similarity function over the knowledge graph
- θ_similarity is the similarity threshold above which two questions
  are considered equivalent (a hyperparameter)

**On the choice of σ:**

The semantic similarity function σ must compare questions against
the structure of Gₜ — not just surface string similarity.
Two questions that use different words but point to the same
missing edge in G are semantically equivalent for this purpose.

The formal definition of σ is an open research problem.
See Section 11, item 2.

## 10. Gap Classification Function

The connection function φ determines how a new chunk c relates
to the current knowledge graph Gₜ:

```
φ(c, Gₜ):

    candidates ← semantic_match(c, Gₜ)

    if candidates = ∅:
        return NO_CONNECTION

    coverage ← compute_why_coverage(c, candidates)

    if coverage ≥ θ_full:
        return FULL_CONNECTION
    else:
        return PARTIAL_CONNECTION
```

Where:
- `semantic_match` returns the set of existing nodes in Gₜ most
  semantically related to c
- `compute_why_coverage` assesses whether the WHY chains of those
  nodes fully explain c
- θ_full is the coverage threshold for a full connection
  (a hyperparameter to be determined experimentally)

**Coverage is not binary:**

The distinction between FULL_CONNECTION and PARTIAL_CONNECTION
is inherently graded. The threshold θ_full is a design parameter
without a principled derivation in this specification.

This is an open problem. See Section 11, item 3.

## 11. Open Problems in This Specification

These are not limitations of Sapien — they are gaps in this
formal specification that require further work before the
specification can be called complete.

**1. Teacher failure and retry limits**
The current specification does not define what happens when a
teacher's answer is rejected by the verifier and human supervisor.
Does the question remain in KU indefinitely? Is there a retry limit?
Is the question archived as unresolvable? No formal answer currently
exists.

**2. Formal definition of semantic similarity σ**
The uniqueness check requires a function σ over the knowledge graph.
This function must be semantically meaningful — comparing graph
structure, not just strings. The exact definition of σ depends on
the graph embedding approach chosen for the implementation, which
has not yet been specified.

**3. Coverage threshold θ_full**
The boundary between PARTIAL_CONNECTION and FULL_CONNECTION is
threshold-dependent. θ_full is a hyperparameter that must be
empirically determined. Setting it too high means the learner
always generates questions — never reaching closure. Setting it
too low means it accepts shallow understanding as complete. The
correct value is domain-dependent and may not be universal.

**4. Chunk granularity**
This specification assumes T partitions τ into well-formed chunks.
It does not specify what a chunk is, how large it should be, or
how the partitioning is performed. If chunks are too large, partial
connections produce too many gaps simultaneously. If too small,
the episode becomes computationally intractable. Chunk size is an
open engineering and pedagogical question.

**5. Convergence across episodes**
This specification defines closure within a single episode.
It does not formally define convergence across multiple episodes —
how L's knowledge graph approaches complete coverage of a domain.
A multi-episode convergence theorem remains unwritten.
